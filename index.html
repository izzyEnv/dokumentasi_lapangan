Tentang Proyek

Proyek ini bertujuan untuk membuat tampilan login hotspot Mikrotik yang diintegrasikan dengan payment gateway Mayuni untuk pembelian voucher paket internet dan lainnya.

Komponen yang Digunakan

  • Router Mikrotik
  • VPS Ubuntu 22.04
  • Domain Publik

Teknologi yang Digunakan

  • Node Js (javascript)
  • Docker Container
  • Service Cloudflare

flow project

saat pengguna terhubung ke hotspot Mikrotik dan diarahkan ke halaman login. Pengguna memilih paket, membayar via payment gateway Mayuni, dan setelah berhasil, server akan memberikan voucher melalui API Mikrotik untuk memberikan akses internet.

Integrasi dengan Payment Gateway Mayuni

untuk melakukan integrasi dengan payment gateway Mayuni kamu perlu membuat akun di Mayuni.co.id untuk mendapatkan kredensial api,seperti token dan signature key yang akan digunakan untuk menggunakan service Mayuni.

setelah kamu mendapatkan kredensial api nya,kamu bisa mencoba untuk membuat request transaksi melalui api menggunakan kode javascript simple seperti ini: 


let token = "you're token";
let signature_key = "you're signature_key";
let hostname = "https://api.mayuni.co.id";
let path = "/transaction/create/qris";
let merchant_ref = "TRX83453080";
let amount = 10000;
let expired_duration = 24;
let method = "POST";


const crypto = require("crypto");
function md5(s) {
  return crypto.createHash("md5").update(s)
  .digest("hex");
}

let signature = md5(path + method + merchant_ref + amount + expired_duration + signature_key);

let payload = {
    method: "qris",
    merchant_ref: merchant_ref,
    amount: amount,
    expired_duration: expired_duration,
    customer_name: "customer name",
    customer_phone: "085536495868",
    customer_email: "customer@gmail.com",
    signature: signature,
    callback_url: "https://merchant.co.id/callback"
};

const fetchData = async () => {
    try {
        const res = await fetch(hostname + "/api/v1/mayuni-partner" + path, {
            method: "POST",
            headers: {
                "Content-Type": "application/json",
                "token": token
            },
            body: JSON.stringify(payload)
        });

        const data = await res.json(); 
        console.log(data); 
        return data;

    } catch (error) {
        console.error("Gagal:", error);
        return error;
    }
};

fetchData(); // panggil fungsi

Parameter Request Transaksi

Berikut adalah detail parameter yang dibutuhkan untuk membuat request transaksi.

method
Metode pembayaran yang digunakan, misalnya "qris".
merchant_ref
ID transaksi unik yang Anda buat.
amount
Nominal transaksi dalam satuan angka (contoh: 10000).
expired_duration
Durasi kadaluarsa transaksi dalam satuan jam.
customer_name
Nama lengkap pelanggan.
customer_phone
Nomor telepon pelanggan.
customer_email
Alamat email pelanggan.
signature
Kombinasi hash MD5 dari beberapa parameter.
Formula: path + method + merchant_ref + amount + expired_duration + signature_key
callback_url
URL yang akan menerima notifikasi status transaksi (berhasil/gagal) dari Mayuni.

untuk dokumentasi lengkapnya cek di dokumentasi mayuni

Untuk parameter penting seperti method dan amount, aku menggunakan atribut `data-*` pada elemen HTML untuk menyimpan nilainya dan mengambilnya menggunakan javascript.

  • Parameter `amount`: aku menambahkan harga langsung pada voucher.
    ... data-amount="10000" ...
  • Parameter `method`: untuk method aku tempatkan pada tombol pilihan pembayaran sesuai dengan tampilannya .
    <button data-method="qris">QRIS</button>

    • selebihnya bisa di sesuaikan dengan preferensi masing masing

Konfigurasi VPS

pastikan vps sudah terhubung/satu jaringan dengan mikrotik,aku menggunakan Node.js dan Docker untuk menjalankan library routerOS-client dan Cloudflare Tunnel.

Langkah-langkah

  1. Instalasi
    Untuk menginstall Docker dan Node.js. kamu dapat mengikuti panduan instalasi dari web resminya dari sini :
    • Docker Docs
    • Node.js
    Tinggal copy paste doang kok
  2. Inisialisasi project
    Buat direktori baru untuk project, masuk ke directory, dan jalankan perintah berikut
    npm init -y
  3. Instalasi Dependensi
    Selanjutnya, install semua library yang dibutuhkan (routerOS-client, cors,express)
    npm install routeros-client cors express --save
  4. Buat File Server
    Buat file JavaScript (misalnya, server.js) dan kamu bisa mencoba kode ini: 
    const express = require('express');
const RouterOSClient = require('routeros-client').RouterOSClient;
const cors = require('cors');
const app = express();
const PORT = 3000;

app.use(cors());
app.use(express.json());

// Konfigurasi koneksi ke Mikrotik
const apiConfig = {
    host: "10.10.10.1",
    user: "api",
    password: "api123"
};

// Endpoint untuk mendapatkan daftar user hotspot
app.get('/api/users', async (req, res) => {
    const api = new RouterOSClient(apiConfig);
    try {
        const client = await api.connect();
        const users = await client.menu('/ip/hotspot/user').getAll();
        await api.close();
        res.json(users);
    } catch (err) {
        console.error("Error fetching users:", err);
        res.status(500).json({ error: err.message || "Internal Server Error" });
    }
});

app.listen(PORT, () => {
    console.log(`Server is running on http://localhost:${PORT}`);
});

    Kode di atas akan mengambil semua data user hotspot yang ada di mikrotik,lalu menaruhnya pada endpoint /api/users.

    kalian bisa menyesuaikan apiConfigdengan informasi login Mikrotik kalian.
    5. Menjalankan & Menguji Server
    Untuk menjalankan server, gunakan perintah berikut di terminal Anda:
    node server.js

    Setelah server berjalan, Anda dapat menguji endpoint API dengan perintah `curl`:

    curl http://localhost:3000/api/users
    Ini hanya salah satu contoh penggunaan sederhananya, kalian bisa kustomisasi sesuai dengan kebutuhan kalian,seperti menambahkan header untuk keamanan atau apapun yang kalian inginkan. Untuk detailnya kalian bisa cek di RouterOS Client Wiki.

Konfigurasi docker container

untuk menjalankan containernya aku membuat 1 dockerfile dan docker-compose untuk memudahkan proses build dan run nya:

Dockerfile: 
# node.js version
FROM node:22-alpine
# Set working directory
WORKDIR /usr/app
# Copy package.json
COPY package*.json ./
# Install dependencies
RUN npm install --production
# Copy the rest of the app
COPY . .
# Expose express default port
EXPOSE 3000
# Start the server
CMD ["node", "server.js"]
Docker Compose: 
version: '3.8'
services:
  mikrotikapp:
    container_name: mikrotikapp
    build:
      context: .
      dockerfile: dockerfile
    ports:
      - "3000:3000"
    networks:
      - mikrotiknetwork
    restart: unless-stopped

networks:
  mikrotiknetwork:
    name: mikrotiknetwork
    driver: bridge

Dengan konfigurasi ini, Dockerfile akan menyalin semua modul yang diperlukan dan menjalankan server pada port 3000 di dalam container.

docker-compose.yml akan membuat kontainer bernama mikrotikapp dan menggunakan port 3000, lalu menghubungkannya ke jaringan mikrotiknetwork.

Untuk memulainya, jalankan perintah:
docker compose up -d.

Konfigurasi Cloudflare Tunnel

1. Buat Tunnel

Login ke dasboard Cloudflare, navigasi ke menu Zero Trust, lalu pilih Networks > Tunnels.

  1. Klik Create a tunnel.
  2. Pilih tipe tunnel Cloudflared.
  3. Beri nama tunnel.
  4. Pilih environment yang digunakan. Karena aku menggunakan Docker, pilih Docker.
  5. copy perintah Docker yang diberikan dan jalankan di VPS , lalu hubungkan ke jaringan yang sama dengan kontainer `mikrotikapp`.

2. Route Traffic

Selanjutnya, konfigurasikan rute untuk tunnel Anda. Tambahkan domain publik (dan subdomain jika perlu), service HTTP, dan arahkan URL ke nama container beserta port yang digunakan (contoh: `mikrotikapp:3000`).

Setelah semua konfigurasi selesai, data seharusnya sudah bisa diakses melalui nama domain dengan path yang telah dikonfigurasi,bahkan sudah otomatis HTTPS dari cloudflare nya.

tentu saja kalian bisa memodifikasi/mengubah konfigurasinya sesuai dengan keinginan/kebutuhan kalian

Mendapatkan SSL Certificate

Untuk mendapatkan SSL certificate, aku menggunakan Certbot dengan plugin "python3-certbot-dns-cloudflare" yang diinstal di VPS dan API Token dari Cloudflare.

API Token dapat dibuat di dasboard Cloudflare. ke menu Manage Account > API Tokens, lalu pilih "Create Token". Gunakan template "Edit zone DNS" dan arahkan ke domain kalian.

Langkah-langkah :

  1. Masuk ke terminal VPS.

  2. Instalasi Certbot:

    sudo apt install certbot python3-pip -y

  3. Install plugin Cloudflare:

    sudo apt install python3-certbot-dns-cloudflare -y

  4. Buat folder dan file untuk menyimpan token Cloudflare:

    mkdir -p ~/.secrets
nano ~/.secrets/cloudflare.ini

  5. Isi file seperti ini:

    dns_cloudflare_api_token = TOKEN_CLOUDFLARE

  6. Lalu simpan dan atur permission file:

    chmod 600 ~/.secrets/cloudflare.ini

  7. Request certificate:

    sudo certbot certonly \
  --dns-cloudflare \
  --dns-cloudflare-credentials ~/.secrets/cloudflare.ini \
  -d yourdomain.net -d www.yourdomain.net \
  --preferred-challenges dns-01

  8. Renewal certificate otomatis :

    sudo certbot renew --dry-run

setelah itu kamu akan mendapatkan certificate nya di

/etc/letsencrypt/live/yourdomain.net

seperti fullchain.pem dan privkey.pem

untuk dokumentasi lengkapnya bisa cek di: Certbot DNS Cloudflare

Konfigurasi Mikrotik

SSL & DNS Hotspot

  1. 1. salin file certificate fullchain.pem yang sudah didapat sebelumnya ke MikroTik (via Winbox).:

  2. Import certificate di MikroTik:

    /certificate import file-name=fullchain.pem

  3. Aktifkan SSL untuk mengaktifkan HTTPS di Hotspot:

    /ip service set www-ssl certificate=fullchain.pem disabled=no
/ip hotspot profile set [find where name="(nama_profile)"] login-by=https ssl-certificate=fullchain.pem

Walled Garden Bypass

Gunakan fitur Walled Garden supaya tetap bisa akses ke domain VPS sebelum user login hotspot.

Tambahkan wallet garden baru di IP > Hotspot > Walled Garden dengan parameter berikut:

  • Action: allow
  • Dst. Host: namadomainvps.net

*Ganti `namadomainvps.net` dengan nama domain publik VPS.

cara lain, Konfigurasi Domain dengan Nginx(opsional)

Selain menggunakan Cloudflare, sebenarnya bisa juga menggunakan Nginx untuk mengatur domain secara local.

1. Instalasi Nginx

Pastikan Nginx sudah terinstal di vps. Jika belum, gunakan perintah berikut: 

sudo apt update
sudo apt install nginx

2. Buat File Konfigurasi Server

Buat file konfigurasi baru di direktori /etc/nginx/sites-available/: 

sudo nano /etc/nginx/sites-available/namafile

3. Konfigurasi nginx

Tambahkan konfigurasi berikut ke dalam file tersebut, sesuaikan namadomain.com dan proxy_pass yang sesuai dengan port service yang berjalan 

server {
    listen 80;
    server_name namadomain.com www.namadomain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name namadomain.com ;

    ssl_certificate /path/ke/file/fullchain.pem;
    ssl_certificate_key /path/ke/file/privkey.pem;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

4. Aktifkan Konfigurasi dan Restart Nginx

Buat symbolic link untuk mengaktifkan konfigurasi, lalu restart Nginx agar perubahan diterapkan: 

sudo ln -s /etc/nginx/sites-available/namafile /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

Setelah itu,kita perlu menambahkan sedikit konfigurasi di MikroTik.

Konfigurasi DNS

Kita perlu menambahkan DNS static di MikroTik agar domain dapat resolve ke VPS:

  1. Buka menu IP > DNS, lalu pilih Static.
  2. Klik tombol + (Tambah) untuk membuat static DNS baru.
  3. Isi parameter berikut:
    • Name: Masukkan nama domain yang Anda gunakan di VPS (misalnya, yourdomain.net).
    • Address: Masukkan IP VPS.
  4. Klik OK
